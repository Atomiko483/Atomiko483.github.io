Proficiency with Web Development Tools

HTML and CSS Learning Process

I made sure to practice HTML5 and CSS3 extensively without the use of assitive technology like intellisense or tailwind CSS. This was to make sure that even if every tool was taken away from me, so long as I had a place to write code I would be able to replicate any piece of work I could do using frameworks or assistance. I have since then learned Tailwind CSS as it is a useful skillset to have, but I consistenly practice my CSS and HTML to this very day. Even this website was coded using no frameworks, I made all the CSS myself.

Team Collaboration with Git and Github

One of the first steps I took in my web development journey after learning HTML and CSS was to learn basic Git and Github as well as best practices. I recognize the importance of working in a team and being able to share ideas with minimal conflict, so I considered it an essential habit and skillset to pick up early even without having worked in a team. Being able to work well with others is one of my strongest skills, and I intend to continue learning about Version Control Systems like Git in the near future.

Design and Coding Principles

Semantic HTML

While the use of a div or span for styling is acceptable, I never use them to implement features (like a button) that have a semantic tag built into HTML. Instead, I prefer to use these tags so that assitive technology can use them, and then style them however I want using CSS.

Descriptive CSS

Speaking of CSS, if I am not using Tailwind CSS for some reason or another, I make sure that my CSS class names follow the naming convention that is agreed upon by the team I am working with (Block-Element-Modifier naming convention for example). In my own personal work, I tend to use general styles similair to Tailwind CSS such as "flex" or "rotate-90". This websites styling is done in a mix of BEM and my personal style to showcase both naming conventions.
